# Garity 🚗
> **Find Quality Cars Without the Hustle.**

Garity is a comprehensive used-car aggregator and search platform for Kenya. It pulls live listings from platforms like Jiji and standardizes the data into a sleek, fast, and easy-to-use Next.js frontend, allowing users to browse, compare prices, and view detailed information efficiently.

![Garity Preview](/public/images/car-1.png) *(UI preview reference)*

## 🛠️ Tech Stack

- **Frontend:** Next.js 15 (App Router), React, Tailwind CSS, Lucide Icons, Recharts (for price sparklines).
- **Backend/Scraper:** Python 3.13, HTTPX (fast requests), Playwright (anti-bot bypass & JS rendering), BeautifulSoup4 (HTML parsing).
- **Data Storage:** Local JSON file generated by the scraper (served via Next.js API route).

## 🚀 Getting Started

To run Garity locally, you'll need to set up both the **Next.js Web App** and the **Python Scraper**.

### Prerequisites
- Node.js 18+
- Python 3.10+

### 1. Setup the Next.js Web App

Clone the repository and install the frontend dependencies:

```bash
git clone https://github.com/yourusername/garity.git
cd garity

# Install dependencies
npm install

# Start the development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser. By default, if no live scraped data exists, the app will gracefully fall back to mock data.

### 2. Setup the Python Scraper

The Python scraper fetches real-world car listings from Jiji Kenya, extracts high-resolution image galleries, and parses detailed car attributes (like Transmission, Color, Interior Color, etc.).

Open a new terminal in the `garity` project root:

```bash
# Navigate to the scraper directory
cd scraper

# Create a virtual environment
python -m venv .venv

# Activate the virtual environment
# On Windows:
.venv\Scripts\activate
# On macOS/Linux:
source .venv/bin/activate

# Install required Python packages
pip install -r requirements.txt

# Install Playwright browsers (required for the scraper's browser-fallback mechanism)
playwright install chromium
```

### 3. Run the Scraper

To fetch live data, run the scraper script from the **root of the project** (`garity` directory), using the python executable from your virtual environment.

**Basic Run (1 Page):**
```bash
scraper\.venv\Scripts\python.exe -m scraper.run
```
*(On macOS/Linux use `scraper/.venv/bin/python -m scraper.run`)*

**Advanced Run (Multiple Pages & Detail Enrichment):**
```bash
scraper\.venv\Scripts\python.exe -m scraper.run --pages 3 --max-details 15
```

**CLI Arguments:**
- `--pages <int>`: Number of search result pages to scrape (default: 1).
- `--max-details <int>`: Max number of cars to enrich by visiting their dedicated detail pages (default: 10). Detail pages yield higher-res images and extra attributes.
- `--skip-details`: Skip visiting detail pages entirely (faster, but less data).
- `--output <path>`: Custom path to save the JSON output.

Once the scraper finishes, it will save the data to `scraper/data/listings.json`. The Next.js API route (`/api/listings`) automatically detects this file and serves the live data to the frontend immediately!

## 📁 Project Structure

```text
garity/
├── src/
│   ├── app/                # Next.js App Router (Pages & API routes)
│   ├── components/         # Reusable React components (CarGrid, ImageCarousel, etc.)
│   ├── data/               # Fallback mock data
│   └── utils/              # Helper functions (e.g., formatPrice)
├── scraper/
│   ├── parsers.py          # BeautifulSoup logic for Jiji HTML extraction
│   ├── scraper.py          # HTTPX and Playwright request handling
│   ├── run.py              # CLI entry point for the Python scraper
│   └── data/               # Generated listings.json output directory
├── public/                 # Static assets and images
└── next.config.mjs         # Next.js config (configured for Jiji external images)
```
